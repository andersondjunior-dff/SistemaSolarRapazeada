Descodificando o Nosso Projeto!

Olá, pessoal! Hoje vamos apresentar como criámos o nosso Sistema Solar Interativo.

Usámos código que parece complexo, mas que na verdade funciona com regras muito lógicas e divertidas.

Foco da Apresentação: Vamos explicar como o site se adapta a celulares e computadores (Media Screen) e como funciona o botão de destruir planetas!

O que é o "Media Screen"?

Imaginem que o nosso site é um astronauta. Ele precisa vestir roupas diferentes dependendo do planeta onde vai pousar!

O Media Screen (ou Media Query) no CSS é exatamente isso. Ele diz ao navegador:

"Se a tela for um monitor gigante de computador, mostra o Sistema Solar grande e o painel ao lado. Se for um celular pequeno, encolhe tudo e deixa proporcional!"

Tela de Computador

Como a tela Funciona no Código?

No CSS do nosso projeto, criámos uma regra matemática simples baseada na largura da janela:

@media screen and (max-width: 768px) significa:
"Se a largura da tela for menor que 768 píxeis (celulares)..."
  • Reduzimos a escala global (zoom out) do Sistema Solar.
  • Mudamos o painel de dados para deslizar como uma gaveta do fundo da tela.
@media screen and (max-width: 768px) {
  .solar-system {
    /* Encolhe o tamanho do mapa inteiro */
    --responsive-scale: 0.35;
  }
  .info-card {
    /* Posiciona o painel no rodapé */
    bottom: -100%;
    width: 100%;
  }
}

O Mistério do Botão de Destruição

Como funciona a destruição passo a passo? Nós criámos um **plano de ataque** ordenado no JavaScript!

Temos uma lista escrita (um Array) com a ordem de quem deve explodir primeiro (de fora para dentro):

const ordemRemocao = ['Plutão', 'Netuno', 'Urano', 'Saturno', 'Júpiter', 'Cintura', 'Marte', 'Terra', 'Vénus', 'Mercúrio'];

Cada vez que clicamos no botão, o código procura quem é o próximo da lista e inicia um processo em cadeia.

O que o Código Faz ao Clicar?

Ao carregar no botão, o JavaScript executa uma sequência rápida de 4 passos:

  • 1. Treme a tela: Aplica a animação shake ao tela para sentirmos o impacto físico da explosão.
  • 2. Ativa o fogo: Adiciona a classe CSS .explode ao planeta, fazendo-o brilhar e crescer imenso.
  • 3. Espera 0.6s: Usa um temporizador (setTimeout) para deixar a animação visual rodar.
  • 4. Apaga do HTML: Remove permanentemente o planeta da página usando o comando alvo.remove().
// 1. Treme a tela
shakeContainer.classList.add('shake');

// 2. Ativa a animação de explosão
planeta.classList.add('explode');

// 3. Espera a animação acabar (600ms)
setTimeout(() => {
    // 4. Apaga do site para sempre!
    alvo.remove();
}, 600);

Dava para fazer SEM JavaScript?

Como ainda não aprendemos JavaScript a fundo, esta é a pergunta mais importante! Seria possível fazer isto apenas com HTML e CSS?

A resposta é NÃO! E aqui está a explicação simples:

  • O HTML e o CSS são como um Cenário de Teatro: Eles desenham os planetas e fazem-nos girar de forma bonita no piloto automático.
  • O JavaScript é o Diretor da Peça: Sem ele, o botão de destruir seria apenas um desenho inútil. Precisamos do JS para "ouvir" o clique, contar quantos planetas restam e apagá-los um a um!

HTML + CSS (O Cenário)

Apenas cria o visual estático e as órbitas rotativas contínuas.

JavaScript (A Ação)

Ouve o clique, executa a contagem, faz a Supernova acontecer e altera o site.

O Gran Finale: A Supernova!

Quando a nossa lista de planetas chega a zero, o JavaScript deteta que já não há mais nada para destruir.

Então, ele muda o texto do botão para "ALERTA DE SUPERNOVA!" e desativa-o.

Após 1.2 segundos, o Sol expande-se 40 vezes o seu tamanho original e pinta o fundo da tela de amarelo brilhante!
